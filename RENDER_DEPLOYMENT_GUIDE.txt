# 🚀 Kvasir Dergi - Render.com Deployment Kılavuzu

## 📋 ÖN GEREKSINIMLER

## 🎯 RENDER.COM'DA DEPLOYMENT ADIMLARı

### 1. RENDER.COM'A GİRİŞ
- https://render.com adresine gidin
- "Sign Up" ile hesap oluşturun (GitHub ile bağlanabilirsiniz)
- Dashboard'a erişin

### 2. YENİ WEB SERVICE OLUŞTURMA
- Dashboard'da "+ New" butonuna tıklayın
- "Web Service" seçin
- "Build and deploy from a Git repository" seçin
- "Connect" butonuna tıklayın

### 3. GITHUB REPOSITORY BAĞLAMA
- GitHub hesabınızı bağlayın (izin verin)
- Repository listesinde "kvasir-dergi" yi bulun
- "Connect" butonuna tıklayın

### 4. SERVICE KONFİGÜRASYONU
Aşağıdaki ayarları yapın:

**Basic Settings:**
- Name: kvasir-dergi
- Region: Oregon (US West) [veya size en yakın]
- Branch: main
- Root Directory: (boş bırakın)

**Build & Deploy:**
- Runtime: Node
- Build Command: npm ci && npm run build
- Start Command: npm start

**Advanced Settings - Environment Variables:**
NODE_ENV = production
NEXT_TELEMETRY_DISABLED = 1

**Instance Type:**
- Free (0.5 GB RAM, 0.5 CPU) - Test için yeterli
- Starter ($7/ay) - Production için önerilen

### 5. DEPLOYMENT BAŞLATMA
- "Create Web Service" butonuna tıklayın
- İlk deployment otomatik başlayacak (5-10 dakika sürer)

### 6. DEPLOYMENT DURUMU TAKİBİ
Build log'larında şunları göreceksiniz:
```
==> Installing dependencies...
==> Building Next.js application...
==> Starting application...
==> Your service is live at https://kvasir-dergi.onrender.com
```

## 🔧 DEPLOYMENT SONRASI AYARLAR

### CUSTOM DOMAIN (İsteğe Bağlı)
1. Service Settings'e gidin
2. "Custom Domains" bölümünde kendi domain'inizi ekleyin
3. DNS kayıtlarını güncellemeyi unutmayın

### ENVIRONMENT VARIABLES (Gelecekte ihtiyaç halinde)
- DATABASE_URL (PostgreSQL için)
- NEXTAUTH_SECRET (Auth için)
- EMAIL_SERVER_HOST (Email için)

### AUTO-DEPLOY AYARI
- Settings > "Auto-Deploy" açık olsun
- GitHub'a her push'ta otomatik deploy olur

## 🚨 ÖNEMLİ NOTLAR

### Free Tier Limitleri:
- ⏰ 15 dakika inaktiviteden sonra sleep mode
- 💾 500GB bandwidth/ay
- 🕐 İlk erişimde 30-60 saniye açılış süresi (cold start)

### Performance Tips:
- Starter plan ($7/ay) ile 24/7 aktif kalır
- Static dosyalar için CDN kullanımı önerilir
- Database için PostgreSQL plugin eklenebilir

### Troubleshooting:
- Build failed ise: Dependencies eksik olabilir
- Memory error ise: Starter plan'a geçin
- 502 error ise: Start command'i kontrol edin

## 🚨 502 BAD GATEWAY DETAYLI ÇÖZÜM

### ADIM 1: Environment Variables Kontrol
Render.com Dashboard > Settings > Environment Variables:
- NODE_ENV = production
- PORT = 10000
- NEXT_TELEMETRY_DISABLED = 1

### ADIM 2: Build & Deploy Settings Kontrol
- Build Command: npm install && npm run build
- Start Command: npm start

### ADIM 3: Debug Logs Kontrol
Deploy logs'ta şunları arayın:
✅ "npm install" başarılı mı?
✅ "npm run build" başarılı mı?
✅ "npm start" çalışıyor mu?
✅ "ready - started server on 0.0.0.0:10000" mesajı var mı?

### ADIM 4: Manuel Müdahale
Eğer hala çalışmıyorsa:
1. Render.com'da "Manual Deploy" yapın
2. Logs sekmesinde gerçek zamanlı logları izleyin
3. Build sırasında hata var mı kontrol edin

### ADIM 5: Alternative Start Command
Eğer npm start çalışmıyorsa, Render.com'da Start Command'i şöyle değiştirin:
- `./node_modules/.bin/next start -H 0.0.0.0 -p $PORT`

### ADIM 6: Node.js Version Kontrol
Render.com'da Node.js 18+ kullandığınızdan emin olun:
- Dashboard > Settings > Environment Variables
- NODE_VERSION = 18 (eğer yoksa ekleyin)
